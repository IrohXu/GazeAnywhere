Clinical Gaze Intelligence with AR Glasses

GazeTrack Agent is a doctor-facing system for pediatric gaze studies. It combines AR voice interaction, ASR, SAM3 timeline understanding, and a Claude Agent core to turn recorded sessions into structured, queryable clinical insights.

Overview

Built for Pediatric Gaze Study in Real Clinical Workflows

The product focus is practical adoption: clinicians can define profiles, capture AR recordings, and ask temporal gaze questions without dealing with model internals. The system is service-oriented, explicit about lifecycle control, and designed to integrate into research and hospital-side pipelines.

1. Model

GazeAnywhere as the Perception Backbone

GazeAnywhere powers gaze target estimation from recording data and provides the primary perception outputs used downstream. This keeps the analysis stack focused on practical inference and robust replay behavior.

  • Foundation gaze estimation for unconstrained scenes
  • Serviceized for deployment and independent scaling
  • Produces profile-level outputs for temporal querying
2-3. Core Stack

ASR + SAM3 + Claude Agent

AR Voice + ASR

Voice commands from AR glasses are transcribed into clean text tasks before agent planning begins.

SAM3 Timeline Layer

SAM3 segmentation builds scene-object timelines so gaze events can be anchored to what exists in the frame at each time point.

Claude Agent Core

Claude Agent handles language understanding, structured task planning, and response generation for clinician-facing interaction.

Controlled Runtime

Agent execution runs in a constrained sandbox with allowlisted tools to prioritize stability and predictable behavior.

4. Functional Scope

What the Current System Can Do

Profile Setup

Create tracking profiles from appearance-based descriptions.

Recording Control

Manage START, frame upload, and STOP via a dedicated channel.

Trajectory Overlay

Generate offline visual overlays for replay and review.

Temporal Query

Ask who/what was gazed at in a selected time range.

Object Window Search

Find when a target object was looked at, with semantic matching.

Behavior Analysis

Analyze gaze switch events and dwell segments across sessions.

5. Interaction

AR Voice to Analysis Workflow

  1. Clinician speaks commands through the AR device interface.
  2. ASR converts speech into a normalized text command.
  3. Claude Agent parses intent and maps it into structured executable tasks.
  4. Execution uses /ws/agent and /ws/recording channels.
  5. Pipeline runs formatter -> codegen -> sandbox execution -> naturalizer.
  6. Artifacts are persisted per profile for replay and later comparison.

9. Team & Project

Real-time "where is the child looking now" live queries are not included in the current release and are tracked as future expansion.

Agentic Gaze Analysis with AR Interaction

Houze Yang  ·  Xu Cao  ·  Chen Fang  ·  Inki Kim  ·  James Matthew Rehg

Core model: GazeAnywhere — accepted to CVPR 2026 (main conference)

Social gaze—where a person looks, and at whom, during face-to-face interaction—provides an important window into social cognition. Atypical gaze behavior is a well-established marker of autism spectrum disorder and related neurodevelopmental conditions, but measuring it rigorously in clinical settings remains challenging. Manual annotation from video recordings is labor-intensive, subjective, and difficult to scale. As a result, objective gaze metrics are still hard to obtain in both clinical practice and behavioral research, especially in naturalistic, real-world settings.

GazeTrack Agent addresses this challenge through an end-to-end platform designed for pediatric clinical studies. During a session, AR glasses continuously capture egocentric video. GazeAnywhere, a promptable gaze model accepted to CVPR 2026, estimates gaze targets in real time from natural-language subject descriptions, avoiding reliance on brittle detection pipelines. In parallel, Qwen and SAM3 construct a semantic scene representation that associates gaze locations with named objects in the environment. Finally, a Claude-based reasoning agent supports open-ended post-session queries through voice interaction, enabling clinicians and researchers to move more directly from recorded behavior to interpretable, actionable insights.

System Architecture
System Pipeline

Real-Time Capture + Post-Session Query

The system operates in two phases. During recording, AR glasses stream egocentric frames to the server where GazeAnywhere runs promptable gaze estimation in real time, while Qwen builds a scene vocabulary in the background. After the session, SAM3 runs on the full video to produce object-level segmentation — because clinically, knowing what the subject was looking at matters as much as knowing where. The Claude agent then handles open-ended voice queries over the resulting gaze-object timeline.

Perception Backbone

GazeAnywhere: Promptable Gaze Target Estimation CVPR 2026

GazeAnywhere introduces Promptable Gaze Target Estimation (PGE) — a new end-to-end, concept-driven paradigm for gaze analysis. Instead of relying on brittle multi-stage pipelines that require explicit head bounding boxes or pose estimation, GazeAnywhere conditions gaze prediction directly on flexible text or visual prompts, integrating subject localization with gaze estimation in a single forward pass.

GazeAnywhere

End-to-End Concept-Driven Gaze Estimation

Given an image and a natural-language prompt like "the boy in the blue shirt", GazeAnywhere simultaneously localizes the subject's head, predicts a 2D gaze target heatmap, and classifies whether the gaze target is in or out of frame.

The architecture uses frozen DINOv3 and dino.txt encoders to extract visual and text features, projected into a shared embedding space. A transformer-based detector fuses these representations and feeds three specialized decoders: a Gaze Tracker (heatmap), a Head Tracker (bounding box), and a Presence Predictor (in/out classification).

Why Promptable?

Prior gaze models require pre-detected head bounding boxes as input, creating a sequential dependency on external detectors (OVDs). Errors in detection cascade into gaze estimation failures — especially in crowded scenes and with children's faces.

GazeAnywhere eliminates this bottleneck. A text prompt like "the boy in the blue shirt" is all the model needs to identify the subject and estimate gaze, end-to-end.

SOTA Results

GazeAnywhere achieves state-of-the-art on all PGE benchmarks: GazeFollow-Concept, VAT-Concept, ChildPlay-Concept, and the out-of-domain Child-SC clinical dataset. It outperforms all two-stage OVD+Gaze pipelines while being faster and more robust.

Text prompting matches visual prompting performance — appearance and pose descriptions are the most critical prompt components.

Agent Design

Multi-Stage LLM Agent with Controlled Execution

Not a single prompt-response call. The agent decomposes each request into a validated task spec, generates executable code against a constrained API surface, runs it in a sandbox, and converts outputs into clinician-facing language.

1

Formatter

Claude Sonnet

Classifies free-form voice input into a structured task spec — task type, subject profile, time range, query targets. Rejects ambiguous requests before computation begins.

2

Code Generator

Claude Sonnet

Translates the validated spec into a Python function that calls only allowlisted perception APIs. No imports, no file access — composable analysis logic only.

3

Sandbox Runtime

Constrained Python

Generated code runs in an isolated namespace. Built-ins blocked; only pre-injected helpers for gaze data, timeline queries, semantic matching, and visualization.

4

Naturalizer

Claude Haiku

Converts structured results into a readable sentence for the clinician — closing the loop from voice question to spoken answer.

Interaction Design

AR Glasses as the Primary Interface

The clinician wears AR glasses throughout the session, keeping full attention on the child. Recording is started and stopped via a physical button on the device — deliberately avoiding voice for this step, so the clinician can prepare without disturbing the patient.

All analysis commands are issued by voice. Spoken queries are transcribed server-side via ASR and routed to the agent. After the session, the same interface supports replay queries, trajectory review, and behavioral analysis.
